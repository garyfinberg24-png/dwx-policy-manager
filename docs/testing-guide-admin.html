Role Overview & Navigation

ⓘ
Who is the Admin?
The Admin role (Tier 4) is the highest access level in Policy Manager. It is assigned to HR Administrators, IT Admins, Executives, and Site Admins. Admins have all capabilities from every lower tier plus: the full Admin panel with 21 configuration sections, the Quiz Builder for creating quizzes, system configuration, and the ability to manage all policies regardless of ownership.

Admin Panel — Layout & Navigation

🚀
Navigate to: PolicyAdmin.aspx
The Admin Panel uses a sidebar + content area layout. The 280px collapsible sidebar lists all configuration sections grouped into categories. Clicking a section loads its settings in the main content area.

Admin Sidebar Sections

Configuration

  • Templates
  • Metadata Profiles
  • Approval Workflows
  • Compliance Settings
  • Notifications
  • Naming Rules
  • SLA Targets
  • Data Lifecycle
  • Navigation

Management

  • Reviewers & Approvers
  • Audit Log
  • Data Export

The sidebar contains 12+ sections across two categories: Configuration (system settings) and Management (operational tools). Each section loads a distinct settings panel in the content area to the right.

The sidebar is 280px wide with a #f1f5f9 background. The active section is highlighted with a teal light background (#ccfbf1).

ADM-02 Admin Panel Sidebar Navigation P1 Critical
Verify the admin sidebar loads all sections and clicking each one switches the content area correctly.
  1. 1
    Open the Admin panel
    Navigate to PolicyAdmin.aspx via the Settings cog or direct URL.
  2. 2
    Verify the sidebar layout
    The left sidebar should be 280px wide with category headings (Configuration, Management) and clickable section items below each. The first section should be auto-selected.
  3. 3
    Click through each sidebar section
    Click every section in the sidebar, one at a time. For each, verify the content area updates to show the corresponding settings panel. There should be no blank/empty panels or JavaScript errors.
  4. 4
    Verify active state styling
    The currently selected section should be highlighted with a teal light background. Previously selected sections should return to normal styling.
✓ Expected Results
  • Sidebar displays all sections grouped into categories
  • Clicking a section updates the content area
  • Active section highlighted in teal
  • No blank panels or errors on any section

Navigation Toggles

ⓘ
How Navigation Toggles Work:
The Navigation section in the Admin panel allows admins to show/hide specific nav items for all users. Toggles are persisted to pm_nav_visibility in localStorage. The PolicyManagerHeader reads this key and filters nav items accordingly. This provides a quick way to disable features that aren't ready or aren't needed.
ADM-03 Toggle Navigation Items On/Off P1 Critical
Test toggling nav items on/off and verify the changes take effect in the header navigation immediately.
  1. 1
    Navigate to Admin → Navigation section
    Click "Navigation" in the admin sidebar.
  2. 2
    Verify all toggle items are present
    You should see toggle switches for at least these items:
    • Policy Hub
    • My Policies
    • Policy Builder
    • Policy Author
    • Policy Packs
    • Distribution
    • Policy Manager
    • Analytics
    • Quiz Builder
  3. 3
    Toggle OFF "Analytics"
    Click the toggle switch for Analytics to disable it. It should change to the "off" state.
  4. 4
    Verify the change persists
    Refresh the page. The toggle should still be off. Check the header nav — "Analytics" should no longer appear.
  5. 5
    Use "Enable All" button
    Click "Enable All" to restore all nav items. Verify all toggles switch to on and all nav items reappear.
  6. 6
    Test "Disable All" button
    Click "Disable All". All toggles should switch to off. Verify the header nav shows no items (or only protected/essential items).
  7. 7
    Re-enable all and confirm
    Click "Enable All" to restore everything.
✓ Expected Results
  • All 9 toggle items present with on/off switches
  • Toggling off hides the item from header navigation
  • Changes persist in pm_nav_visibility localStorage
  • "Enable All" restores all items; "Disable All" hides all
  • Changes visible immediately in header (no page reload needed, or after save)

  1. 1

    View existing naming rules

    • Separator: "-"
    • Counter: 4-digit auto-increment

  4. 4
    Preview the generated name
    A preview should show the resulting pattern, e.g., "POL-0001-2026".
  5. 5
    Save the rule
    Save and verify it appears in the rules list.
✓ Expected Results
  • Naming rules displayed as cards with segment chips
  • New rule creation with configurable segments works
  • Live preview shows the generated name pattern
  • Rules save and persist

  5. 5

✓ Expected Results
  • Audit log displays entries from PM_PolicyAuditLog
  • Each entry shows timestamp, event type, user, policy, and details
  • Filters narrow results correctly
  • Recent test actions (create, approve, publish, acknowledge) are recorded
  • Entries are sorted by most recent first

Quiz Builder

🚀
Navigate to: QuizBuilder.aspx
The Quiz Builder is an Admin-exclusive tool for creating quizzes that accompany policies. It supports 11 question types, AI-powered question generation (via Azure OpenAI), and detailed quiz configuration including scoring, time limits, and certificates.
ADM-09 Create a Quiz with Manual Questions P1 Critical
Test creating a new quiz, adding questions of different types, configuring settings, and publishing the quiz.
  1. 1
    Open the Quiz Builder
    Click "Quiz Builder" in the nav bar. The page should load showing existing quizzes (or empty state) and a "Create Quiz" button.
  2. 2
    Create a new quiz
    Click "Create Quiz". Enter:
    • Title — e.g., "Data Protection Policy Quiz"
    • Description — brief description
    • Associated Policy — select a published policy
    • Passing Score — e.g., 70%
    • Max Attempts — e.g., 3
    • Time Limit — e.g., 15 minutes (or unlimited)
  3. 3
    Add a Multiple Choice question
    Click "Add Question". Select type "Multiple Choice". Enter:
    • Question text
    • 4 options (A-D)
    • Mark the correct answer
    • Add explanation text
    • Set points (e.g., 10)
  4. 4
    Add a True/False question
    Add another question with type "True/False". Enter question text and mark the correct answer.
  5. 5
    Add a Multiple Select question
    Add a third question with type "Multiple Select". Enter question text, options (with multiple correct answers marked), and points.
  6. 6
    Save the quiz as Draft
    Save the quiz. It should appear in the quiz list with "Draft" status and 3 questions.
  7. 7
    Publish the quiz
    Click "Publish" on the draft quiz. Status should change to "Published". The quiz is now available for assignment to policies.
✓ Expected Results
  • Quiz creation form has all required fields
  • Multiple question types can be added
  • Each question type has its specific input fields
  • Quiz saves as Draft with correct question count
  • Publish changes status to Published
  • Records created in PM_PolicyQuizzes and PM_PolicyQuizQuestions
ADM-10 AI Question Generation P2 High
Test the AI-powered question generation feature that uses Azure OpenAI to generate quiz questions from policy document content.
⚠
Pre-requisite: The Azure Function dwx-pm-quiz-func-prod must be deployed and functional. The function key must be configured. A policy with substantial text content must exist.
  1. 1
    Open the Quiz Builder and select "AI Generate"
    In the Quiz Builder, look for an "AI Generate" or "Generate with AI" panel/button.
  2. 2
    Configure generation settings
    • Source Policy — select a policy with rich text content
    • Number of Questions — e.g., 5
    • Difficulty Level — Easy, Medium, or Hard
    • Question Types — select desired types (Multiple Choice, True/False, etc.)
  3. 3
    Click "Generate"
    The system sends the policy text to the Azure Function, which calls Azure OpenAI GPT-4o. A loading indicator should appear.
  4. 4
    Review generated questions
    After a few seconds, generated questions should appear. Review each for relevance, accuracy, and formatting. Questions should be contextually relevant to the policy content.
  5. 5
    Import selected questions
    Select the questions you want to keep and import them into the quiz. They should appear alongside any manually added questions.
✓ Expected Results
  • AI generation panel accepts policy selection and configuration
  • Azure Function responds with structured JSON questions
  • Generated questions are relevant to the policy content
  • Questions can be reviewed before importing
  • Imported questions appear in the quiz with correct structure
  • If Azure Function is unavailable, a clear error message displays (not a crash)

✓ Expected Results
  • Archive action available for Published policies
  • Confirmation required before archiving
  • Status changes to Archived, IsActive = false
  • Policy no longer appears in active Employee views
  • Audit log records the archive event
