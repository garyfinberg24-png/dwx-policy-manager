Azure OpenAI (GPT-4o) is the largest variable cost, shared across three features:
  • Quiz Generator: ~2K-5K tokens/generation ($0.01-0.05). 100 quizzes/month ≈ $5-10.

All three can be individually enabled/disabled in Admin Centre. When disabled, cost = $0.

Cost Optimisation Notes
  • Azure Free Tier: Application Insights (5 GB/month), Function Apps (1M executions/month), and Storage (5 GB) all include generous free tiers that cover typical usage.
  • Reserved Instances: Not applicable — all resources use Consumption/Pay-As-You-Go tiers with no minimum commitment.

3. Premium Option — Azure Database (Instead of SP Lists)

For organisations requiring higher performance, complex queries, or >5,000 list items (SharePoint list view threshold), an Azure database can replace SharePoint Lists as the data store. The SPFx webparts would connect via an API layer (Azure Functions) instead of PnP/SP direct list queries.

Recommendation: Azure SQL Database (Basic or Standard S1)
Azure SQL Basic ($5/month) is the recommended starting point for most organisations. It provides:
  • No 5,000-item list view threshold (SP Lists limitation)
  • Full SQL queries with JOINs, aggregations, and complex reporting
  • Built-in backups (7-day point-in-time restore)
  • TLS encryption in transit + Transparent Data Encryption at rest
  • Azure AD authentication (same Entra ID as M365)
  • Upgrade to S1 ($30/month) when concurrent users or data volume grows
Migration path: An API layer (Azure Functions) would sit between the SPFx webparts and Azure SQL, replacing the current PnP/SP list queries. The 30+ SharePoint lists would map to ~20 normalised SQL tables. Estimated migration effort: 40-60 hours.
When to Stay on SharePoint Lists
  • <5,000 policies (within list view threshold)
  • No complex cross-table reporting needs
  • Small team (<50 concurrent users)
  • Want zero additional Azure cost
  • Don't need SQL-level audit compliance
When to Move to Azure SQL
  • >5,000 policies or acknowledgement records
  • Complex compliance reporting (JOINs, pivots)
  • 100+ concurrent users
  • Data residency/sovereignty requirements
  • Need SQL-level backups, auditing, encryption
  • Planning multi-app DWx suite data sharing
