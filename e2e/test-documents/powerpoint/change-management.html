Change Types

Standard

Pre-approved, low-risk, routine changes. No CAB approval needed.

Example: Patch deployment, user account changes, certificate renewals

Normal

Non-urgent changes requiring planning and CAB approval. 5-day lead time.

Example: Application upgrade, firewall rule change, new integration

Emergency

Urgent fix for P1/P2 incidents. Expedited approval by Emergency CAB.

Example: Security patch for active exploit, production hotfix

2 / 4

Change Request Requirements

Every change request must include:

  • Description: Clear explanation of what is changing and why
  • Impact Assessment: Systems affected, users impacted, downtime expected
  • Risk Assessment: Likelihood and impact of failure
  • Rollback Plan: Step-by-step procedure to revert the change if issues arise
  • Test Evidence: Results from testing in non-production environment
  • Implementation Plan: Detailed steps, schedule, and responsible parties
  • Communication Plan: Who needs to be notified, when, and how

Change Advisory Board (CAB)

Governance and approval

CAB meets weekly (Wednesday 2pm) to review and approve Normal change requests.

Members: CTO (Chair), IT Operations Manager, Security Lead, Application Owners, Service Desk Manager

Change Windows

  • Standard maintenance: Tuesday and Thursday 10pm - 2am AEST
  • Major releases: Saturday 6am - 12pm AEST
  • Change freeze: 15 December - 5 January (annual), plus company-defined blackout periods
