Deployment & Configuration Guide

0 Prerequisites

Required Before Deployment
  • Microsoft 365 tenant with SharePoint Online (E3/E5 or SharePoint Plan 2)
  • SharePoint Admin access (App Catalog, site creation)
  • Azure Subscription with Contributor role
  • Azure CLI installed (v2.50+) with Bicep
  • Node.js 18.x, 20.x, or 22.x
  • PnP PowerShell module installed (Install-Module PnP.PowerShell)
  • Global Admin or Application Admin for Entra App Registration
Build Tools (Development Machine)
  • Node.js 18.17.1+ / 20.x / 22.x
  • npm 9+ or pnpm
  • Gulp CLI: npm install -g gulp-cli
  • SPFx Yeoman Generator: npm install -g @microsoft/generator-sharepoint
  • ~2 GB free disk space for webpack production build

1 Create SharePoint Site

1.1 — Create the Site Collection

Create a Communication Site in SharePoint Admin Centre or via PowerShell:

# Connect to SharePoint Admin
Connect-PnPOnline -Url https://TENANT-admin.sharepoint.com -Interactive

# Create the site
New-PnPSite -Type CommunicationSite `
  -Title "Policy Manager" `
  -Url https://TENANT.sharepoint.com/sites/PolicyManager `
  -Owner "admin@TENANT.onmicrosoft.com"

1.2 — Provision SharePoint Lists (30+ lists)

Run the master provisioning script which creates all PM_ lists with correct columns and indexes:

# Connect to the Policy Manager site
Connect-PnPOnline -Url https://TENANT.sharepoint.com/sites/PolicyManager -Interactive

# Run master deployment (all numbered scripts)
cd scripts/policy-management
.\Deploy-AllPolicyLists.ps1
Idempotent Scripts
All provisioning scripts check for existing lists/fields before creating. Safe to re-run.

# Create all application pages

2 Build & Deploy SPFx Package

2.1 — Install Dependencies & Build

cd policy-manager
npm install

# Production build
gulp clean
gulp bundle --ship
gulp package-solution --ship

# Output: sharepoint/solution/policy-manager.sppkg (11 MB)

2.2 — Upload to App Catalog

  1. Go to SharePoint Admin Centre → More features → Apps → App Catalog
  2. Click "Apps for SharePoint"
  3. Upload policy-manager.sppkg
  4. Check "Make this solution available to all sites in the organization"
  5. Click Deploy
Important: Trust Dialog
The trust dialog will show 17 webpart components. Click "Enable app" to allow all webparts to run on the tenant.

2.3 — Add Webparts to Pages

For each page, edit it and add the corresponding webpart from the "DWx Policy Manager" group. Set each page to Full-width column layout.

3 Deploy Azure Infrastructure

3.1 — Quiz Generator (AI)

cd azure-functions/quiz-generator/infra
.\deploy.ps1 -Environment prod -Location swedencentral

Deploys: Function App + Azure OpenAI + Key Vault + Storage + App Insights

3.2 — Chat Assistant (AI)

cd azure-functions/policy-chat/infra
.\deploy.ps1 -Environment prod -Location swedencentral

Deploys: Function App + Storage + App Insights (reuses OpenAI + Key Vault from quiz-rg)

3.3 — Email Sender (Logic App)

cd azure-functions/email-sender/infra
.\deploy.ps1 -Environment prod

Deploys: Logic App (Consumption) + Office 365 API Connection + SharePoint API Connection

Manual Step Required
After deployment, go to Azure Portal → Logic App → API Connections and authorize both the Office 365 and SharePoint connections. This requires interactive OAuth consent and cannot be automated via Bicep/ARM.

3.4 — Distribution Processor

# Deploy distribution queue processor
cd azure-functions/distribution-processor/infra
.\deploy.ps1 -Environment prod

4 Post-Deployment Configuration

4.1 — Seed Sample Data (Optional)

cd scripts/policy-management
.\Deploy-SampleData.ps1
.\Seed-ApprovalAndNotificationData.ps1

4.2 — Admin Centre Configuration

Navigate to PolicyAdmin.aspx and configure:

  • Templates — Upload policy document templates
  • Metadata Profiles — Create compliance presets (Risk, Timeframe, Ack)
  • Approval Workflows — Configure approval chains
  • Compliance Settings — Set risk levels and review frequencies
  • Notifications — Enable/disable email notification types
  • Naming Rules — Configure policy number format (e.g. POL-{CAT}-{NUM})
  • SLA Targets — Set acknowledgement and review SLA days
  • Navigation — Toggle which nav items are visible per role
  • AI Assistant — Enable AI Chat, set Function URL + API key
  • Theme — Customise colours, logo, fonts via Theme Editor

5 Verification Checklist

Post-Deployment Smoke Test

  • Start.aspx shows the Start Screen with sidebar + quick actions
  • PolicyHub.aspx shows the Policy Hub with published policies
  • MyPolicies.aspx shows the user's assigned policies
  • PolicyBuilder.aspx shows the wizard (Standard/Fast Track)
  • Create a test policy → Save Draft → verify in pipeline
  • Submit for Review → check reviewer receives notification email
  • Approve the policy → verify status changes
  • Publish → verify it appears in Policy Hub
  • PolicyAdmin.aspx loads with sidebar sections
  • PolicySearch.aspx search returns results
  • PolicyHelp.aspx shows all 5 tabs
  • PolicyAnalytics.aspx shows all 6 tabs
  • Footer shows correct build number on all pages
  • Theme Editor changes are reflected across all pages + Start Screen

Email Pipeline Verification

  • Logic App is running (Azure Portal → Logic App → Overview → Enabled)
  • API connections authorized (Office 365 + SharePoint)
  • Create policy → Submit for Review → email arrives in Outlook within 5 min
  • Admin > System Info > Email Pipeline Health shows "Healthy"
  • No items with empty RecipientEmail in PM_NotificationQueue

Playwright E2E Tests (Optional)

# Run the full smoke test suite
cd policy-manager
npx playwright test e2e/pages-deep.spec.ts --headed

# Run full lifecycle test
npx playwright test e2e/step-by-step.spec.ts --headed
