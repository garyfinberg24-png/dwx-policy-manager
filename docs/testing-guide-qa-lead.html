Status Transitions

  • Draft → In Review: Author clicks "Submit for Review" (requires at least 1 reviewer assigned)
  • In Review → Approved: ALL assigned reviewers approve (triggers approval notification to approvers if any)
  • In Review → Draft: Reviewer requests changes OR rejects (all reviewer statuses reset)
  • Approved → Published: Author clicks "Publish" on pipeline (version bumps to next major)
  • Published → Retired: Author retires with reason (outstanding acks cancelled, users notified)
  • Published → Draft (Revision): Author clicks "Revise" (version snapshot, original stays published)

Document Conversion

  • Office docs (DOCX, PPTX, XLSX) are converted to HTML at submit-for-review time
  • PDFs are NOT converted — displayed via native browser PDF viewer
  • HTML policies are rendered natively in the reader (no iframe)
  • Safety-net fallback conversion also runs at publish time if missed
