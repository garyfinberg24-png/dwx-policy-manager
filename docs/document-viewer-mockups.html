Data Protection & Privacy Policy

This policy establishes the principles, guidelines, and requirements for the protection of personal and sensitive data across all business operations at First Digital.

1. Purpose

The purpose of this policy is to ensure that all employees understand their responsibilities regarding the collection, processing, storage, and disposal of personal data in compliance with applicable privacy legislation.

2. Scope

This policy applies to all employees, contractors, and third-party service providers who access, process, or store personal data on behalf of the organisation.

  • All departments and business units
  • All information systems that process personal data
  • Both electronic and physical records containing personal information

3. Data Classification

Classification Description Examples Handling
Restricted Highly sensitive personal data Health records, financial data, TFN Encrypted, access-controlled, audit logged
Confidential Internal business data with personal elements Employee records, client contacts Need-to-know access, secure storage
Internal Business information not for public Meeting notes, project plans Standard access controls
Public Information approved for release Marketing material, job postings No restrictions

4. Key Principles

Personal data must be processed lawfully, fairly, and transparently. Collection must be limited to what is necessary for the specified purpose.
  1. Lawfulness — Process data only with a lawful basis
  2. Purpose limitation — Use data only for specified purposes
  3. Data minimisation — Collect only what is necessary
  4. Accuracy — Keep data accurate and up to date
  5. Storage limitation — Retain only as long as needed
  6. Integrity — Protect against unauthorised access

5. Document Pipeline Overview
How documents flow from author upload to reader rendering.
1
Author Uploads Document
Author attaches .docx or .pdf to the policy in the wizard. File stored in PM_PolicySourceDocuments.
2
Author Publishes Policy
PolicyService.publishPolicy() fires. If DocumentURL is a .docx, triggers conversion.
3
Azure Function Converts
.docx path: Downloads from SharePoint → mammoth.js converts to HTML → Forest Teal CSS applied → HTML returned
.pdf path: No conversion needed — skipped
4
HTML Saved to PolicyContent
Converted HTML stored in PM_Policies.PolicyContent field. One-time conversion — not repeated on each read.
5
Reader Renders
Has PolicyContent? → Native HTML reader (fast, scrollable, printable, themed)
No PolicyContent + .pdf? → Native browser PDF embed (no iframe wrapping)
No PolicyContent + .docx? → Office Online iframe viewer (fallback)
