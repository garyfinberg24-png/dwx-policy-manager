Password Security

Strong Password Rules

  • Minimum 14 characters — longer is stronger
  • Mix of uppercase, lowercase, numbers, and symbols
  • Never reuse passwords across accounts
  • Use a password manager (approved: 1Password)
  • Enable MFA on all accounts — no exceptions

Common Mistakes

  • Using personal information (birthdays, pet names)
  • Writing passwords on sticky notes
  • Sharing credentials with colleagues
  • Using the same password for work and personal
  • Ignoring MFA prompts you did not initiate
MFA Fatigue Attacks: Never approve an MFA prompt you did not initiate. Report unexpected prompts to IT Security immediately.

Recognising Phishing Attacks

Red Flags to Watch For

  • Urgency: "Your account will be locked in 24 hours"
  • Spoofed sender: ceo@f1rstdigital.com (note the '1' instead of 'i')
  • Suspicious links: Hover before clicking — does the URL match?
  • Unexpected attachments: Especially .exe, .zip, .js files
  • Requests for credentials: IT will never ask for your password via email
  • Grammar and spelling errors: Professional organisations proofread

DO

  • Report suspicious emails via the Phish Alert button
  • Verify requests via a separate channel
  • Check the actual URL before clicking

DON'T

  • Click links in unexpected emails
  • Download attachments from unknown senders
  • Reply with sensitive information

Protecting Company Data

Data Classification Levels

Restricted
Board papers, M&A, credentials
Confidential
HR records, financials, client data
Internal
Policies, procedures, org charts
Public
Marketing materials, job ads

Key Rules

  • Classify all documents before sharing
  • Never store Restricted or Confidential data on personal devices
  • Use approved file-sharing tools only (SharePoint, OneDrive)
  • Encrypt sensitive attachments before emailing externally
  • Lock your screen when leaving your desk (Win+L)

Secure Remote Working

Home Office Security

  • Use company VPN for all work activities
  • Secure your home Wi-Fi with WPA3 encryption
  • Keep work and personal devices separate
  • Use a privacy screen in public spaces
  • Shred physical documents containing sensitive data

Public Spaces

  • Never connect to free public Wi-Fi for work
  • Use mobile hotspot if office Wi-Fi unavailable
  • Be aware of shoulder surfing
  • Do not take calls with sensitive content in public
  • Never leave devices unattended

Reporting Security Incidents

If you suspect a security incident, report it immediately. Early reporting reduces impact.

What to Report

  • Lost or stolen devices (laptop, phone, USB drive)
  • Suspected phishing emails or social engineering attempts
  • Unusual system behaviour or unexpected pop-ups
  • Accidental data exposure or misdirected emails
  • Suspicious physical access to secure areas

Social Engineering Awareness

Attackers manipulate people, not just technology. Be vigilant against these tactics:

Common Techniques

  • Pretexting: Impersonating IT support or a vendor
  • Baiting: Leaving infected USB drives in common areas
  • Tailgating: Following authorised staff into secure areas
  • Vishing: Phone-based phishing scams
  • Quid pro quo: Offering something in exchange for information

Defence Strategies

  • Always verify identity before sharing information
  • Use callback verification for financial requests
  • Never plug in unknown USB devices
  • Challenge unfamiliar visitors in secure areas
  • Trust your instincts — if something feels wrong, report it

Key Takeaways

Remember these five principles every day

  1. Think before you click — Verify links, senders, and requests
  2. Protect your credentials — Strong passwords + MFA everywhere
  3. Classify and protect data — Handle information according to its classification
  4. Report immediately — Early reporting minimises impact
  5. Stay vigilant — Security is everyone's responsibility
