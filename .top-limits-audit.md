## Risk Assessment

### High Risk - .top(5000) Performance Concerns:
1. **PolicyHubService** - 5 calls to .top(5000) for dashboard metrics calculations
2. **PolicyReportExportService** - 8 calls to .top(5000) for bulk report generation
3. **PolicyAuditService** - Loading entire audit log with .top(5000)
4. **PolicyRetentionService** - Querying all policies for retention rules
5. **PolicyPackService** - Loading all pack assignments at once

**Recommendation**: Implement pagination, filtering, or use SharePoint search instead of loading 5000+ items.

### Medium Risk - .top(1000):
- Multiple analytics queries could cause performance degradation
- Recommend implementing pagination or reducing selected columns

### Best Practice:
- Use `.top(` with specific filters to reduce results
- Use `.orderBy()` with `.top()` to get relevant items first
- Consider pagination for large datasets
- Use field selection to reduce payload size

## Example: PolicyHubService.ts Line 82

```typescript
// Context (Lines 80-87):
const policyQuery = this.sp.web.lists.getByTitle(this.POLICIES_LIST).items;

// Get total count before pagination
const allPolicies = await policyQuery.top(5000)();
const totalCount = allPolicies.length;

// Filter to active policies
const activePolicies = allPolicies.filter(p => p.PolicyStatus === PolicyStatus.Published);
```

**Issue**: Loads ALL 5000 policies just to count/filter active ones. Should use filter in query instead.

**Better approach**:
```typescript
const activePolicies = await this.sp.web.lists
  .getByTitle(this.POLICIES_LIST)
  .items.filter(`PolicyStatus eq '${PolicyStatus.Published}'`)
  .top(5000)();
```
