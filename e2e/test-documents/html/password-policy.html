Password and Authentication Policy

Policy ID: FD-IS-005 | Version 6.0 | Effective: 1 January 2026

1. Purpose

This policy defines the requirements for creating, managing, and protecting passwords and authentication credentials across all First Digital systems. Strong authentication is a critical defence against unauthorised access and is essential for protecting the organisation's data, systems, and reputation.

2. Scope

This policy applies to all accounts on all systems owned, operated, or managed by First Digital, including user accounts, service accounts, administrative accounts, and API keys. It applies to all employees, contractors, and third-party users.

3. Password Requirements

3.1 Standard User Accounts

RequirementStandard
Minimum Length14 characters
ComplexityMust include at least 3 of: uppercase, lowercase, numbers, special characters
Maximum Age365 days (annual rotation required)
HistoryCannot reuse the last 12 passwords
LockoutAccount locked after 5 consecutive failed attempts for 30 minutes

3.2 Privileged and Administrative Accounts

RequirementStandard
Minimum Length20 characters
ComplexityMust include all 4 character types
Maximum Age90 days
HistoryCannot reuse the last 24 passwords
Additional ControlPrivileged Access Management (PAM) vault required; just-in-time access

3.3 Service Accounts and API Keys

  • Service account passwords must be a minimum of 30 characters, randomly generated.
  • Service accounts must not have interactive login capability unless specifically required and approved.
  • API keys must be stored in a secrets management solution (e.g., Azure Key Vault); never in source code or configuration files.
  • API keys must be rotated every 180 days or immediately upon suspected compromise.

4. Multi-Factor Authentication (MFA)

Multi-factor authentication is mandatory for all users across all First Digital systems. The following MFA methods are supported, in order of preference:

  1. FIDO2 Security Keys (hardware tokens) - preferred for administrators and high-risk users.
  2. Microsoft Authenticator App (push notification or TOTP) - standard for all employees.
  3. SMS one-time passcode - permitted only as a temporary fallback; must be replaced within 14 days.

MFA must be enforced on all external-facing applications, VPN connections, cloud administration consoles, and any system processing Restricted or Confidential data.

5. Prohibited Practices

  • Sharing passwords with any other person, including IT staff (IT will never ask for your password).
  • Writing passwords on paper, sticky notes, or storing them in unencrypted files.
  • Using the same password across multiple systems (password reuse).
  • Using personal passwords for business accounts or vice versa.
  • Sending passwords via email, instant message, or other unencrypted channels.
  • Using dictionary words, personal information, or predictable patterns (e.g., "Summer2026!").

6. Password Managers

First Digital provides a corporate password manager (1Password Business) to all employees. Use of the corporate password manager is strongly recommended for generating and storing unique, complex passwords for each application. Personal password managers may be used for personal accounts only and must not store corporate credentials.

7. Compromised Credentials

If an employee suspects their password has been compromised, they must immediately change the password and notify the SOC. The IT Security team monitors for credential exposure in data breaches via dark web monitoring services. Employees identified in breach databases will be required to reset their password within 24 hours.

8. Responsibilities

RoleResponsibility
All UsersCreate strong passwords; enable MFA; never share credentials; report compromises.
IT OperationsEnforce password policies via Azure AD / Group Policy; deploy MFA; monitor for anomalies.

9. Compliance

Non-compliance with this policy will result in forced password reset and may result in disciplinary action. Repeated violations or deliberate credential sharing will be treated as a serious security breach.
