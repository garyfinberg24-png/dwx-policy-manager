Role Overview & Navigation

ⓘ
Who is the Employee?
The Employee role (internally called "User") is the default access tier for all staff members in the organisation. Every person who logs into the Policy Manager site automatically receives this role unless they are additionally assigned Author, Manager, or Admin privileges via SharePoint group membership. Employees are the primary consumers of policies — they browse, read, acknowledge, and complete quizzes on assigned policies.

What the Employee Sees

When an Employee opens the Policy Manager site, the header navigation bar shows only the items permitted for their role. Author, Manager, and Admin features are hidden.

Navigation Item Page Visible? Purpose
Policy Hub PolicyHub.aspx ✓ Yes Central browsing, categories, recently viewed
My Policies MyPolicies.aspx ✓ Yes Personal dashboard with assigned policies
Policy Details PolicyDetails.aspx ✓ Yes Read policy content, acknowledge, take quiz
Policy Builder PolicyBuilder.aspx ✗ Hidden Author+ only
Policy Author PolicyAuthor.aspx ✗ Hidden Author+ only
Policy Packs PolicyPacks.aspx ✗ Hidden Author+ only
Distribution PolicyDistribution.aspx ✗ Hidden Manager+ only
Analytics PolicyAnalytics.aspx ✗ Hidden Manager+ only
Quiz Builder QuizBuilder.aspx ✗ Hidden Admin only

Header Action Buttons

Action Visible? Notes
🔍 Search ✓ Yes Opens search center or inline search
🔔 Notifications ✓ Yes In-app notification bell
❔ Help ✓ Yes Opens help center
⚙ Settings (Cog) ✗ Hidden Manager+ only — should NOT appear for employees
EMP-01 Verify Role-Based Navigation Visibility P1 Critical
Confirm that an Employee-level user sees only the permitted navigation items and that restricted pages are not accessible, even by direct URL.
  1. 1
    Log into the Policy Manager site as an Employee user
    Use a test account that is not a member of any elevated SharePoint groups (Authors, Managers, Admins). The account should only have standard site member permissions.
  2. 2
    Navigate to the Policy Hub page
    https://mf7m.sharepoint.com/sites/PolicyManager/SitePages/PolicyHub.aspx
  3. 3
    Inspect the header navigation bar
    Count the navigation items. You should see only: Policy Hub, My Policies. Optionally also the Search, Help, and Notifications icons.
  4. 4
    Confirm hidden items are NOT present
    Verify these items do not appear: Policy Builder, Policy Author, Policy Packs, Distribution, Analytics, Manager View, Quiz Builder.
  5. 5
    Confirm Settings cog is NOT visible
    The gear/cog icon should not appear in the header action area. Only Search, Notifications, and Help icons should be present.
  6. 6
    Attempt direct URL access to restricted pages
    Try navigating directly to each restricted page by entering the URL. The application should either redirect you or show limited/empty content:
    .../SitePages/PolicyBuilder.aspx — Author page
    .../SitePages/PolicyAdmin.aspx — Admin page
    .../SitePages/PolicyAnalytics.aspx — Manager page
✓ Expected Results
  • Navigation bar shows only Policy Hub and My Policies
  • Header shows Search, Notifications, Help — but not the Settings cog
  • Direct URL navigation to restricted pages either shows a "not authorised" message, an empty/blank webpart, or the page loads without the elevated features
  • No JavaScript console errors related to permission denial

Policy Hub — Central Browsing

🚀
Navigate to: PolicyHub.aspx
The Policy Hub is the main landing page for policy discovery. It provides category browsing, recently viewed policies, featured policies, and quick search functionality.
EMP-02 Policy Hub Page Load & Layout P1 Critical
Verify the Policy Hub loads correctly, displays the Forest Teal branding, and shows the main content sections.
  1. 1
    Open the Policy Hub page
    Navigate to PolicyHub.aspx. The page should load within a few seconds, showing a loading spinner initially if data is being fetched.
  2. 2
    Verify the page header and branding
    The header should display the teal gradient (#0d9488 → #0f766e), the Policy Manager logo/title, and the navigation bar. Breadcrumbs should show: Policy Manager / Policy Hub.
  3. 3
    Check the main content sections
    The page should display these sections:
    • Featured Policies — highlighted/promoted policies
    • Browse by Category — category cards with counts
    • Recently Viewed — your recently opened policies (may be empty on first visit)
    • Quick Search — search input for policy lookup
  4. 4
    Verify content area layout
    Content should be centred with a max width of 1400px, with 24px padding on each side. Cards should be properly spaced and aligned.
✓ Expected Results
  • Page loads without errors; loading spinner appears briefly then content renders
  • Forest Teal gradient header is visible
  • All content sections are present and properly laid out
  • No "undefined" or "null" text visible in any section
EMP-03 Browse Policies by Category P2 High
Test category-based browsing to ensure policies are grouped correctly and clickable.
  1. 1
    Locate the category cards section
    Look for "Browse by Category" or similar heading. Categories may include: HR, IT, Finance, Compliance, Health & Safety, etc.
  2. 2
    Click on a category card
    Click on any category (e.g., "HR Policies"). The view should filter to show only policies in that category.
  3. 3
    Verify the filtered results
    Check that only policies tagged with the selected category are shown. Each policy card should display: title, category tag, status, version, and a brief description.
  4. 4
    Click on a policy to open it
    Click "View" or the policy title. This should navigate you to PolicyDetails.aspx with the policy ID as a URL parameter.
✓ Expected Results
  • Category cards display with names and policy counts
  • Clicking a category filters the policy list correctly
  • Policy cards in filtered view show accurate metadata
  • Clicking a policy navigates to PolicyDetails.aspx
EMP-04 Recently Viewed Tracking P3 Medium
Verify that the "Recently Viewed" section updates after viewing policies and persists across page loads (stored in localStorage).
  1. 1
    Note the current "Recently Viewed" state
    On a fresh browser session, the Recently Viewed section should be empty or show a "No recently viewed policies" message.
  2. 2
    Open a policy from the Hub
    Click "View" on any policy card. Wait for PolicyDetails to load. This triggers RecentlyViewedService.trackView() which stores the policy in localStorage under the key pm_recently_viewed.
  3. 3
    Navigate back to Policy Hub
    Use the navigation bar or browser back button.
  4. 4
    Verify the policy appears in Recently Viewed
    The policy you just viewed should now appear in the Recently Viewed section. It should show the most recently viewed first.
  5. 5
    View 2-3 more policies, then check ordering
    The most recently viewed should always be first. Maximum 10 items are stored.
✓ Expected Results
  • Recently Viewed starts empty on first visit
  • After viewing a policy, it appears in the Recently Viewed section
  • Most recent is shown first; max 10 items displayed
  • Data persists across page reloads (stored in pm_recently_viewed localStorage)

My Policies — Personal Dashboard

🚀
Navigate to: MyPolicies.aspx
The My Policies page is the employee's personal compliance dashboard. It shows all policies assigned to them, their acknowledgement status, due dates, and overall compliance progress.
⚠
Pre-requisite: For meaningful testing, the test account should have at least 3-5 policies assigned via the PM_PolicyAcknowledgements list — including at least one unread, one due soon, and one already acknowledged. Ask an Admin tester to distribute policies to your test account before starting this section.
EMP-05 My Policies Dashboard & Compliance Progress P1 Critical
Verify the personal dashboard loads correctly, shows the compliance percentage, and displays all assigned policies with accurate status indicators.
  1. 1
    Open the My Policies page
    Click "My Policies" in the navigation bar, or navigate directly to MyPolicies.aspx.
  2. 2
    Verify the compliance progress indicator
    At the top of the page, you should see a compliance percentage (e.g., "87% Complete"). This is calculated as: (Acknowledged Policies / Total Assigned) × 100. If you have 5 assigned and 3 acknowledged, it should show 60%.
  3. 3
    Check the tab bar
    You should see 4 tabs:
    • All — all assigned policies
    • Unread — policies not yet opened
    • Due Soon — overdue or due within 7 days
    • Completed — fully acknowledged policies
  4. 4
    Click each tab and verify filtering
    Switch between each tab. The "All" tab should show all policies. "Unread" should only show policies with status "Unread" or "Sent". "Due Soon" should show policies where the due date is within 7 days or past. "Completed" should show only acknowledged policies.
  5. 5
    Inspect a policy card
    Each policy row or card should display:
    • Status badge (coloured): Unread, In Progress, Completed, Overdue
    • Title — clickable policy name
    • Category and Department
    • Version number
    • Due Date — red if overdue
    • Priority — High (red), Medium (orange), Low (green)
    • Actions: View, Acknowledge, Quiz (if required)
  6. 6
    Test search functionality
    Use the search box to filter policies by name. Type part of a policy title — the list should filter in real-time.
  7. 7
    Toggle between list and card view
    If a view toggle exists, switch between list and card view. Both should display the same data in different layouts.
✓ Expected Results
  • Compliance percentage displays and is mathematically correct
  • All 4 tabs work and filter policies correctly
  • Policy cards show all required metadata without "undefined" values
  • Overdue policies display due dates in red
  • Search filters in real-time
  • View toggle switches layout correctly

Policy Details & Acknowledgement

🚀
Navigate to: PolicyDetails.aspx?policyId=<id>
This page displays the full content of a policy and allows the employee to acknowledge it. It is the most critical page for compliance — this is where the employee reads, understands, and formally acknowledges a policy.
EMP-06 View Policy Content & Metadata P1 Critical
Verify the full policy document renders correctly with all metadata, versioning info, and the document viewer.
  1. 1
    Open a policy from My Policies or Policy Hub
    Click on any assigned policy. The page should navigate to PolicyDetails.aspx?policyId=X.
  2. 2
    Verify the policy metadata panel
    The metadata section should display:
    • Policy Title — prominent heading
    • Policy Number — e.g., POL-0001
    • Category & Sub-Category
    • Department / Owner
    • Version — e.g., 2.0
    • Effective Date & Expiry/Review Date
    • Status — Published (green badge)
    • Compliance Risk Level — Critical, High, Medium, Low, or Informational
    • Data Classification — Public, Internal, Confidential, etc.
  3. 3
    Read the policy document content
    The main content area should display the policy body — either as rendered HTML content or an embedded document viewer. If the policy has an attached document (Word/PDF), the document viewer should render it.
  4. 4
    Test the fullscreen document viewer toggle
    If a fullscreen toggle button exists, click it to expand the document viewer to full screen. Click again or press Esc to exit fullscreen.
  5. 5
    Check for related information sections
    The page may also show:
    • Related Policies — links to associated policies
    • Version History — previous versions list
    • Comments/Discussion — if enabled
    • Ratings — policy rating widget
✓ Expected Results
  • All metadata fields display correctly — no "undefined", "null", or blank values
  • Policy content renders as formatted HTML or document embed
  • Fullscreen toggle works correctly
  • Breadcrumbs update to show: Policy Manager / Policy Hub / [Policy Title]
EMP-07 Acknowledge a Policy P1 Critical
This is the most important employee workflow — formally acknowledging a policy. This test verifies the end-to-end acknowledgement process including optional digital signature.
⚠
Important: Acknowledgement is a one-way action. Once acknowledged, it cannot be undone. The acknowledgement record is permanently stored in PM_PolicyAcknowledgements for compliance audit purposes. Use test data for this scenario.
  1. 1
    Open an unacknowledged policy
    From My Policies, select a policy with status "Unread" or "Opened". Navigate to its details page.
  2. 2
    Verify the acknowledgement panel appears
    At the bottom of the policy content, an acknowledgement section should be visible. It typically includes:
    • A statement like "I have read and understood this policy"
    • An "Acknowledge" button
    • Optional digital signature canvas
    • The due date for acknowledgement
  3. 3
    If a digital signature is offered, draw a signature
    Use the signature canvas to draw your name. This captures a base64 image stored in the acknowledgement record.
  4. 4
    Click the "Acknowledge" button
    Click to submit. A confirmation dialog may appear asking you to confirm.
  5. 5
    Verify the acknowledgement success
    After submission, you should see a success message (e.g., "Policy acknowledged successfully"). The acknowledgement panel should update to show "Acknowledged" with the timestamp.
  6. 6
    Return to My Policies and verify the status update
    Navigate back to My Policies. The policy should now appear under the "Completed" tab with a "Completed" status badge. The compliance percentage should increase.
✓ Expected Results
  • Acknowledgement panel visible with statement and button
  • Digital signature (if offered) captures correctly
  • "Acknowledge" button submits successfully with success feedback
  • Policy status changes from "Unread/Opened" to "Acknowledged"
  • Compliance percentage in My Policies updates correctly
  • Acknowledgement record created in PM_PolicyAcknowledgements with date, user, and optional signature
  • If policy requires quiz, user is prompted to take quiz after acknowledgement
EMP-08 Already-Acknowledged Policy Display P2 High
Verify that when an employee revisits a policy they have already acknowledged, the acknowledgement panel shows the completed state rather than offering the option again.
  1. 1
    Open a previously acknowledged policy
    From My Policies → Completed tab, click on a policy you have already acknowledged.
  2. 2
    Check the acknowledgement panel state
    Instead of an "Acknowledge" button, the panel should show: "Acknowledged on [date]" with the timestamp and a green checkmark or similar indicator. The button should be disabled or replaced with a status display.
✓ Expected Results
  • No duplicate acknowledgement option — the button is gone or disabled
  • The acknowledged date and status are displayed clearly
  • The policy content is still fully readable

Quiz Taking

ⓘ
Context: Some policies require employees to pass a quiz after reading the policy to demonstrate comprehension. The quiz system supports 11 question types: Multiple Choice, True/False, Multiple Select, Short Answer, Fill in the Blank, Matching, Ordering, Rating Scale, Essay, Image Choice, and Hotspot. The quiz is typically accessible from the Policy Details page after acknowledgement, or via a direct link.
⚠
Pre-requisite: A published quiz must be associated with at least one policy assigned to the test employee. Ask the Admin tester to create a quiz using the Quiz Builder and assign it to a test policy before starting this section.
EMP-09 Take a Policy Quiz — Full Workflow P1 Critical
Test the complete quiz-taking experience from start to result, including answering different question types, timer behaviour, and score calculation.
  1. 1
    Open a policy that has a required quiz
    In My Policies, look for a policy with a "Quiz" action button or a quiz icon. Click it to navigate to the policy, then look for the quiz prompt.
  2. 2
    Start the quiz
    Click "Start Quiz" or similar button. You should see:
    • Quiz title and description
    • Number of questions
    • Passing score (e.g., 70%)
    • Time limit (if set)
    • Attempt number (e.g., "Attempt 1 of 3")
  3. 3
    Answer questions of various types
    Work through each question. For each type, verify the UI renders correctly:
    • Multiple Choice — 4 radio buttons (A-D), select one
    • True/False — 2 radio buttons
    • Multiple Select — checkboxes, select multiple
    • Short Answer — text input field
    • Fill in the Blank — text input(s) within a sentence
    • Matching — drag-and-drop or dropdown pairs
    • Ordering — drag items into correct sequence
  4. 4
    Monitor the timer (if applicable)
    If a time limit is set, verify the countdown timer displays and counts down. When time runs out, the quiz should auto-submit.
  5. 5
    Submit the quiz
    After answering all questions, click "Submit Quiz" or similar. A confirmation dialog may appear.
  6. 6
    Review the results
    The results page should show:
    • Score — e.g., "8/10 (80%)"
    • Pass/Fail status — based on passing score threshold
    • Time taken
    • Per-question breakdown — if "Show Correct Answers" is enabled
    • Correct/Incorrect feedback — if "Show Explanations" is enabled
    • Retake option — if failed and retakes remain
  7. 7
    If passed — verify certificate (if enabled)
    If the quiz has certificate generation enabled, a "Download Certificate" button should appear. Click it and verify a PDF or printable certificate is generated with your name, quiz title, score, and date.
✓ Expected Results
  • Quiz loads with all questions rendered correctly per their type
  • Timer counts down (if time limit set)
  • Score is calculated correctly after submission
  • Pass/Fail displayed based on passing score threshold
  • If passed: acknowledgement record updated with QuizStatus = Passed
  • If failed + retakes remain: retake option visible with "Attempt 2 of 3" displayed
  • If failed + no retakes: final failure message displayed
  • Certificate downloadable (if enabled and passed)

Policy Search Center

🚀
Navigate to: PolicySearch.aspx
The Search Center provides a dedicated full-page search experience with a hero section, quick-filter chips, sidebar filters, and result cards. It follows the DWx Standard Search pattern used across all Digital Workplace Excellence apps.
EMP-10 Search for Policies Using Keywords & Filters P2 High
Verify the search center returns accurate results for keyword queries and that sidebar filters narrow results correctly.
  1. 1
    Open the Policy Search page
    Click the Search icon in the header, or navigate to PolicySearch.aspx. You should see a hero section with a search input and quick-filter chips below it.
  2. 2
    Perform a keyword search
    Type a known policy name or keyword (e.g., "data protection") in the search box and press Enter or click search. Results should appear below.
  3. 3
    Verify search result cards
    Each result card should display: policy title (clickable), category, department, status, version, effective date, and a brief description or excerpt with the search term highlighted.
  4. 4
    Test quick-filter chips
    Click on a quick-filter chip (e.g., "HR", "IT", "Compliance"). Results should filter immediately to show only policies matching that category.
  5. 5
    Use sidebar filters
    The 260px sidebar filter panel should offer refinements:
    • Category — dropdown or checkboxes
    • Department
    • Status — Published, Archived, etc.
    • Risk Level — Critical, High, Medium, Low
    • Date Range — effective date filter
    Apply a combination of filters and verify results narrow correctly.
  6. 6
    Clear all filters and verify reset
    Click "Clear All" or remove all filters. Results should return to the unfiltered state.
  7. 7
    Click a search result to open the policy
    Click on a policy title in the results. You should be navigated to PolicyDetails.aspx?policyId=X.
✓ Expected Results
  • Hero section renders with search box and quick-filter chips
  • Keyword search returns relevant results
  • Quick-filter chips filter by category instantly
  • Sidebar filters narrow results correctly (AND logic across filter types)
  • Clear All resets to unfiltered state
  • Zero-results state shows a friendly "No policies found" message
  • Clicking a result navigates to the correct policy details page

Help Center

🚀
Navigate to: PolicyHelp.aspx
The Help Center follows the DWx Standard Help pattern — a hero section with tab-based content. It provides articles, FAQs, keyboard shortcuts, and a support request form.
EMP-11 Help Center Navigation & Content P3 Medium
Verify all tabs in the Help Center load content correctly and that the support request form functions.
  1. 1
    Open the Help Center
    Click the Help icon (?) in the header, or navigate to PolicyHelp.aspx.
  2. 2
    Verify the hero section and tab bar
    The page should display a hero section with the title "Help Center" and a tab bar with 5 tabs:
    • Home — overview with getting started cards
    • Articles — help articles
    • FAQs — frequently asked questions (expandable)
    • Shortcuts — keyboard shortcuts reference
    • Support — submit a support request
  3. 3
    Click through each tab
    Click each tab and verify content loads. The "Home" tab should show onboarding cards. "FAQs" should have expandable accordion items. "Shortcuts" should list keyboard bindings.
  4. 4
    Expand an FAQ item
    Click on any FAQ question. The answer should expand smoothly below it.
  5. 5
    Test the Support tab — submit a request
    Switch to the "Support" tab. Fill in the support request form (subject, description, priority). Submit it and verify a success message appears.
✓ Expected Results
  • All 5 tabs render content without errors
  • FAQ accordion expands/collapses smoothly
  • Support form validates required fields and shows success on submit
  • No blank or "coming soon" sections (unless by design)

Notifications

ⓘ
Context: The notification system alerts employees when new policies are assigned, acknowledgement deadlines approach, and quizzes need to be completed. Notifications are delivered both in-app (via the bell icon in the header) and via email.
EMP-12 Notification Bell & In-App Alerts P2 High
Verify the notification bell shows unread count, clicking opens the notification panel, and notifications are accurate.
  1. 1
    Check the notification bell in the header
    Look for the bell icon in the header action area. If there are unread notifications, a badge with the count should appear on the bell.
  2. 2
    Click the notification bell
    A panel or dropdown should appear showing your notifications. Each notification should include: type icon, title/message, timestamp, and priority indicator.
  3. 3
    Click on a notification
    Clicking a notification should navigate you to the relevant page (e.g., a "New Policy Assigned" notification should link to PolicyDetails.aspx for that policy).
  4. 4
    Verify notification types
    Common employee notifications include:
    • "New Policy Published: [Name]" — when a policy is assigned
    • "Acknowledgement Due: [Name] — Due in 3 days" — reminder
    • "Overdue: [Name]" — past due date
    • "Quiz Required: [Name]" — quiz pending after acknowledgement
✓ Expected Results
  • Notification bell shows accurate unread count
  • Panel displays notifications with correct metadata
  • Clicking a notification navigates to the correct destination
  • After viewing, unread count decreases
