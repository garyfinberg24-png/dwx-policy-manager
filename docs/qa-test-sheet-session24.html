Bug Fixes 19

All bugs reported by testers and discovered during E2E testing. Verify each fix in the deployed build.

BugAreaPriorityFix DetailsStatus
Escalation email flooding Notifications Critical EscalationService default changed from enabled: true to enabled: false. Stops the flood of empty-recipient "Review overdue" emails. Admin must explicitly enable via PM_Configuration.
Email pipeline crash — empty recipients Notifications Critical EscalationService now validates recipientEmail.includes('@') before writing to PM_NotificationQueue. ~300 bad items cleaned. Logic App Bicep adds Check_Recipient condition.
Review + Approval emails sent simultaneously Review Policy Critical Submit for Review was combining [...reviewers, ...approvers] into one notification list. Now only sends to reviewers. Approvers notified separately after review approval.
Rejected policy goes back to Draft Review Policy High Reject now sets PolicyStatus: 'Rejected' (was 'Draft'). Request Changes still returns to Draft for editing.
Reject doesn't close remaining reviewer actions Review Policy High When one reviewer rejects, all other pending reviewer statuses set to 'Cancelled' with comment. Previously stayed Pending indefinitely.
Review mode — Office doc content not shown Review Policy High Review mode now shows Office Online viewer (WopiFrame) for .docx/.xlsx/.pptx when no HTML conversion exists. PDF uses <object> embed. Previously showed "No content available".
Role detection — SP groups vs Admin Centre disconnect Roles & Access High RoleDetectionService now reads from PM_UserProfiles.PMRole (Admin Centre) as sole source of truth. SP group detection removed. Custom roles supported.
User Directory rows not clickable Admin Centre High Removed multi-select checkbox mode that intercepted click events. Pencil icon and double-click now open the edit panel.
Policy Packs crash — 'ReviewDate' field missing Policy Packs High Changed 'ReviewDate' → 'NextReviewDate' in PolicyService select clause.
Policy Packs crash — 'Department' field missing Policy Packs High Removed 'Department' from PolicyService select clause — not provisioned on PM_Policies.
Policy Packs crash — 'Visibility' field missing Policy Packs High Removed 'Visibility' from PolicyService select clause.
Search not returning results Policy Hub / Search High applyTextSearch() was checking policy.Description (doesn't exist) instead of PolicyDescription. Now searches across Title, PolicyName, PolicyDescription, PolicyNumber, Category, Risk.
My Policies links throw 404 My Policies Medium Hardcoded /sites/PolicyManager/ URL replaced with dynamic context.pageContext.web.absoluteUrl.
Metadata profile not restored on draft edit Create Policy Medium Draft edit always showed Custom settings instead of the saved profile. Now checks MetadataProfileId first → restores "existing" mode.
Audience step — Targeted/Role options error Create Policy Medium Validation required audience card click even when departments/roles/groups were selected. Now accepts departments OR roles OR security group names.
Start Screen shows on PolicyHub.aspx Policy Hub Medium Start Screen removed from PolicyHub.tsx entirely. Now a standalone webpart (dwxStartScreen) on Start.aspx.
Start Screen sidebar text unreadable Start Screen Medium Opacity-based text → explicit rgba(255,255,255,0.9). SP #F3F2F1 background bleed fixed with transparent !important.
Search placeholder text invisible on gradient Hub / Search / Help Medium Added .pm-hero-search::placeholder CSS rule. Placeholder text now white on gradient backgrounds.
My Policies hero banner unequal padding My Policies Medium Removed margin: '0 -24px' negative margin hack. Equal 40px padding on both sides.
