Design Notes — Changes from Current

  • Hero search placeholder Changed — Placeholder text now rgba(255,255,255,0.6) for readability on teal. Search icon also brightened to match.
  • Hero vertical alignment Changed — All 3 columns (ring+greeting, search, KPIs) now align-items: center instead of flex-end. Search bar centred vertically with the greeting text.
  • Policy # column New — Policy number in teal monospace (e.g. POL-003). Previously inline in subtitle.
  • Version column New — Version number (e.g. v1.5). Previously inline in subtitle.
  • Due Date column New — Due date formatted as "2 Apr 2026". Previously only in detail panel.
  • Category column New — Category (IT Security, HR, etc.). Previously inline in subtitle.
  • Column header row New — Grey header with all 9 columns aligned to the data grid.
  • Eye icon Changed — Replaced chevron with eye icon for "Read Policy". Communicates "click to view/read" more clearly than a directional arrow.
  • Sort order Changed — Default sort: Overdue first (red border), then Pending/Due (by soonest due date), then Completed. Previously unsorted.
  • Completed filter Changed — Now only shows policies with status === 'completed' && acknowledgementDate. Previously showed all "completed" regardless of acknowledgement.
  • Grid layout Changed — 9-column CSS Grid: 44px 1fr 90px 50px 90px 90px auto 100px 36px. All data perfectly aligned between header and rows.
  • Policy name simplified Changed — Policy name is now a single line (no subtitle). Policy #, version, category all have their own columns.
